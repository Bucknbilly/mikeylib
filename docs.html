Mikeyware Library

Full-featured Roblox UI library. Floating windows, tabs, toggles, sliders, dropdowns, color pickers, flag/config system, themes, Lucide icons, layout containers, toast notifications, loading screens, key system, and a built-in HUD — one loadstring.

01 — setup

Installation

Load with loadstring in a LocalScript or executor. Tries CoreGui first, falls back to PlayerGui.

lua
local UILibrary = loadstring(game:HttpGet("https://raw.githubusercontent.com/Bucknbilly/mikeylib/refs/heads/main/mikeylib.lua"))()
02 — example

Quick Start

Create all tabs first, then call switchTab before adding elements to each one.

✖
Tab Rule
Elements always go into the currently active tab. Call win:switchTab(tab) before adding elements. Never add elements right after addTab() without switching first.
1Call win:addTab() for every tab and store each return value.
2Call win:switchTab(tabVar) to activate a tab.
3Add all elements for that tab.
4Repeat 2–3 for every other tab.
5At the end call win:switchTab(firstTab) to set the default visible tab.
lua — example
local UILibrary = loadstring(game:HttpGet("https://raw.githubusercontent.com/Bucknbilly/mikeylib/refs/heads/main/mikeylib.lua"))()
local win = UILibrary.new("My Script", { configName="cfg", theme="Slate" })

local mainTab = win:addTab("Main", "star")
local setTab  = win:addTab("Settings", "settings")

win:switchTab(mainTab)
win:addSection("Combat")
win:addToggle("ESP", false, function(v) print(v) end, "esp")

win:switchTab(setTab)
win:addToggle("Auto-Save", true, function(v) print(v) end)

win:switchTab(mainTab)
win:notify("Loaded", "My Script is ready!")
03 — theming

Themes

Controls accent, background, border, and text colors across the window. Four built-ins: "Dark", "Crimson", "Slate", "Light". Register custom themes with addTheme.

FNUILibrary:addTheme( themeTable )▾

Registers a theme globally. Omitted keys fall back to the Default theme. Must include a name field.

Theme keys
namestringUnique name used with setTheme.
accentColor3Primary highlight (toggles, fills).
backgroundColor3Window background.
surfaceColor3Row/card backgrounds.
borderColor3Border color.
text / textDim / textMutedColor3Primary, secondary, and hint text colors.
toggle / notifBgColor3Toggle active color and notification background.
lua
UILibrary:addTheme({ name="Ocean", accent=Color3.fromRGB(40,160,220), background=Color3.fromRGB(5,12,22) })
UILibrary:setTheme("Ocean")
-- or use per-window: UILibrary.new("Title", { theme = "Crimson" })
✦Apply a theme per-window by passing theme = "Name" in the UILibrary.new() options.
FNUILibrary:setTheme( name )▾

Activates a registered theme by name. No-op if not found. Built-ins: "Dark" "Crimson" "Slate" "Light".

gradient()

Creates a gradient descriptor you can pass as the Color field in addTag() for gradient-colored text.

RETgrad = UILibrary:gradient( stops, options? )▾
Parameters

07 — config & flags

Flag System

Pass a flag string to any element and its value is automatically saved/loaded with saveConfig / loadConfig. Read live values via win.Flags.

NEWFlag system overview▾

Any element that accepts a flag parameter writes its current value into win.Flags[flagKey] whenever it changes. win:saveConfig() serialises the whole table to JSON. win:loadConfig() deserialises it and pushes each value back to the matching element via registered setters.

Supported elements
addToggle / addCheckboxbooleanStored and restored as true/false.
addSlidernumberStored as the snapped numeric value.
addDropdown / addRadioGroupstringStored as the selected string.
addMultiDropdownstring[]Stored as an array of selected strings.
addColorPickerColor3Stored as RGB components.
addTextBoxstringStored as current text.
lua
local win = UILibrary.new("MyScript", { configName = "myconfig" })
win:addToggle("ESP", false, function(v) end, "esp_on")
win:addSlider("Speed", 16, 200, 16, function(v) end, "speed")

-- Read live:
print(win.Flags.esp_on)  -- true / false
print(win.Flags.speed)   -- number
⚠Config is auto-loaded on window creation when configName is set. You do not need to call loadConfig() manually unless you want a manual reload.

saveConfig()

Serialises all flag values to a JSON file in MIKEYWARE_CONFIGS/. Requires configName to be set in UILibrary.new().

RETok = win:saveConfig()▾

Returns true on success, false if no configName was set or write failed. Requires executor filesystem access (writefile).

lua
win:addButton("Save Config", function()
    win:saveConfig()
    win:notify("Config", "Saved!")
end)

loadConfig()

Reads the saved JSON and pushes each value back to the matching UI element. Called automatically on window open when configName is set.

RETdata = win:loadConfig()▾

Returns the raw data table on success, false if no config file exists yet or no configName was set. Calls win:applyFlags() internally to push values to UI controls.

fetchDiscord()

Fetches live Discord server info from the Discord API using the executor's HTTP request. Returns a data table with name, description, member count, online count, icon URL, and the raw API response. Works with a full URL, discord.gg/code, or a bare invite code.

NEWdata = UILibrary.fetchDiscord( inviteCode )▾
Parameters
inviteCodestringBare code ("abc123"), "discord.gg/abc123", or full "https://discord.gg/abc123" URL.
Returns — data table or nil
data.namestringServer name.
data.descriptionstringServer description (empty string if none).

task.spawn(function()

    if data then
        print(data.name)
        print(data.members, "members  ", data.online, "online")
        win:addParagraph(data.name,
            "Members: " .. data.members .. "  Online: " .. data.online, {
            Image     = data.icon,
            ImageSize = 42,
            Buttons   = {{ Title="Join", Callback=function()
                setclipboard(data.inviteUrl)
                win:notify("Discord", "Link copied!", "discord")
            end }},
        })
    end
end)

Key System

Gate the window behind a key prompt. Pass keySystem in UILibrary.new() options. The window is hidden until the validator function returns true.

NEWoptions.keySystem = { title, subtitle, validator }▾
keySystem fields
title optionalstringHeader text on the key dialog. Default "Key Required".
subtitle optionalstringSubtext below the header. Default "Enter the key to continue".

    keySystem = {
        title     = "Key Required",
        subtitle  = "Get your key at discord.gg/example",
        validator = function(k)
            return k == "MY_SECRET_KEY"
        end,
    },
})

local win = UILibrary.new("My Script", {
    keySystem = {
        title     = "Whitelist",
        validator = function(k)
            local res = UILibrary.Request({
                Url    = "https://mysite.com/validate?key=" .. k,
                Method = "GET",
            })
            return res.StatusCode == 200 and res.Body == "valid"
        end,
    },

Loading Screen

Full-screen overlay with a progress bar animation before the window appears. Works standalone or chained after the key system. Pass loadingScreen in UILibrary.new() options.

UPDATEDoptions.loadingScreen = { title, subtitle, duration, bgColor, barColor, ... }▾
Fields
title optionalstringLarge center text. Defaults to the window title.
subtitle optionalstringSmaller text below title. Default "Please wait...".
duration optionalnumberSeconds the progress bar takes to fill. Default 1.2.
bgColor optionalColor3Full-screen background color. Default near-black.
barColor optionalColor3Progress bar fill color. Defaults to the active theme accent.
titleColor optionalColor3Title text color. Default white.
subtitleColor optionalColor3Subtitle text color. Default muted grey.
titleSize optionalnumberTitle font size. Default 26.
subtitleSize optionalnumberSubtitle font size. Default 13.
logo optionalstring | numberLucide name, rbxassetid://, or plain ID. Renders a 64px image above the title.
enabled optionalbooleanSet to false to skip. Default true.
✦When keySystem is also set, the loading screen plays after a successful key entry — not before.
lua — full example
local win = UILibrary.new("My Script", {
    loadingScreen = {
        title         = "My Script",
        subtitle      = "Loading assets, please wait...",
        duration      = 2,
        bgColor       = Color3.fromRGB(6, 6, 10),
        barColor      = Color3.fromRGB(100, 180, 255),
        titleColor    = Color3.fromRGB(255, 255, 255),
        subtitleColor = Color3.fromRGB(120, 140, 180),
        titleSize     = 28,
        subtitleSize  = 13,
        logo          = "star",
    },
})
lua — with keySystem
local win = UILibrary.new("My Script", {
    keySystem = {
        title     = "Key Required",
        validator = function(k) return k == "mykey123" end,
    },
    loadingScreen = {
        title    = "My Script",
        subtitle = "Verified! Loading...",
        duration = 1.5,
    },
})
-- Flow: key screen → correct key → loading animation → window appears

addHUD()

Standalone floating HUD bar — independent of any window. Shows player info, FPS, ping, executor name, game name, and icon buttons for Server Browser, Settings, and a command bar.

NEWhudGui = UILibrary.addHUD( options? )▾

Called as a static method — not on a window instance. Calling it again destroys any previous HUD.

Options
theme optionalstring | tableBuilt-in: "Crimson" "Dark" "Slate". Or pass a custom table overriding any color keys (bar, accent, text, etc.).
position optionalstring"BelowChat" (default) · "TopLeft" · "TopRight" · "BottomLeft" · "BottomRight".
HUD features
Player section—Avatar thumbnail, display name, username, live clock.
Stats section—FPS (color-coded), Ping ms (color-coded), executor name, current game name.
Server Browser—Fetches public servers, tab-filtered (All / Low Ping / Most Players / Few Players), one-click join.
Settings panel—Toggle click/hover sounds, set custom hover sound asset ID. Saved to MIKEYWARE_CONFIGS/MW_HUD_CONFIG.json.
Command bar—Terminal panel with live autocomplete from _G.MWCmds. Tab to autocomplete, Enter to run.
Collapse—Arrow button below/above bar collapses it to hidden. Draggable anywhere on screen.
Returns
hudGui — the ScreenGui. Destroy it to remove the HUD.
lua
-- Standalone HUD, Crimson theme, anchored below chat
UILibrary.addHUD({
    theme    = "Crimson",
    position = "BelowChat",
})

-- Remove it later:
local hud = UILibrary.addHUD({ theme="Slate" })
hud:Destroy()
