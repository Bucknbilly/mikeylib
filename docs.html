Roblox · LocalScript UI Library

Mikeyware Library

A full-featured Roblox UI library. Build floating windows with tabs, toggles, sliders, dropdowns, color pickers, a config/flag system, toast notifications, and a built-in HUD — all loaded in one line.

URL https://raw.githubusercontent.com/Bucknbilly/mikeylib/refs/heads/main/mikeylib.lua
CoreGui Support Tab System Flag / Config System Color Picker Draggable & Resizable Multi-Dropdown Toast Notifications Built-in HUD Server Browser
01 — setup

Installation

Load the library with loadstring in a LocalScript or executor. It attempts to parent to CoreGui first, then falls back to PlayerGui if access is denied.

lua
local UILibrary = loadstring(game:HttpGet("https://raw.githubusercontent.com/Bucknbilly/mikeylib/refs/heads/main/mikeylib.lua"))()
02 — example

Quick Start

The tab system works the same way as most UI libs — create all tabs first, then call switchTab before adding elements to each one.

✖
Tab Rule — Read This First
Elements are always inserted into whichever tab is currently active. Call win:switchTab(tab) before adding elements to any tab. Never add elements directly after addTab() without switching first.
1 Call win:addTab() for every tab you need and store each return value.
2 Call win:switchTab(tabVar) to make a tab active.
3 Add all elements that belong to that tab.
4 Repeat steps 2–3 for every other tab.
5 At the very end, call win:switchTab(firstTab) to set the default visible tab.
lua — quick example
local UILibrary = loadstring(game:HttpGet("https://raw.githubusercontent.com/Bucknbilly/mikeylib/refs/heads/main/mikeylib.lua"))()
local win = UILibrary.new("My Script")

local mainTab     = win:addTab("Main")
local settingsTab = win:addTab("Settings")

win:switchTab(mainTab)
win:addSection("Combat")
win:addToggle("ESP", false, function(v) print(v) end, "esp_enabled")
win:addSlider("Speed", 16, 200, 16, function(v) print(v) end)

win:switchTab(settingsTab)
win:addToggle("Auto-Save", true, function(v) print(v) end)

win:switchTab(mainTab)
win:notify("Loaded", "My Script is ready!")
03 — window

UILibrary.new()

Creates a new floating window. If a window with the same title already exists it is destroyed first, making re-execution safe.

NEW win = UILibrary.new( title?, options? ) ▾

Attempts to parent to CoreGui, falls back to PlayerGui. Window position is remembered between sessions using MIKEYWARE_POSITIONS/ folder. Title bar shows Mikeyware • title.

Parameters
title optional string Shown in the title bar. Also used as the de-duplication key and the position save key.
options optional table Configuration table. See options below.
Options Table
hideKey Enum.KeyCode Key that toggles window visibility. Defaults to RightShift.
discord string If set, the discord link is automatically copied to clipboard on first load (once per session).
configName string If set, enables the config/flag system. Flags are auto-saved and loaded from MIKEYWARE_CONFIGS/configName.json.
obfuscate boolean If true, randomizes the ScreenGui name in the Explorer. Defaults to false.
Returns — window object (win)
win:addTab(name, icon?) — create a tab
win:switchTab(tab) — activate a tab
win:minimize() — toggle collapsed state
win:destroy(skipConfirm?, onDone?) — close the window
win:resize() — recalculate window height
win:notify(...) — show a toast
win:confirm(msg, onConfirm, onCancel) — show confirm dialog
win:onClose(fn) — register a close callback
win:addConnection(signal, fn) — managed connection
win:saveConfig() / win:loadConfig() — config persistence
win.Flags — table of current flag values
All add* element methods
⚠ RightShift toggles visibility by default. Override with options.hideKey.
lua
local win = UILibrary.new("My Script", {
    hideKey    = Enum.KeyCode.RightControl,
    discord    = "https://discord.gg/example",
    configName = "myscript_cfg",
    obfuscate  = true,
})

addTab()

Registers a new tab in the navigation bar. Returns a tab reference required by switchTab() — always store it.

RET tab = win:addTab( name, icon? ) ▾

Creates the tab button and an empty content container. The first tab added is automatically switched to. Tabs scroll horizontally if there are many of them.

Parameters
name string Label shown in the tab button.
icon optional string Emoji or text prefix shown before the tab name. E.g. "⚔️".
Returns — tab ref
An opaque reference. Pass it to win:switchTab(tab).
✖ Never add elements right after addTab() without calling switchTab() first. Elements go into whatever tab was active before.

switchTab()

Makes a tab active. Every add* call after this goes into that tab.

FN win:switchTab( tab ) ▾
Parameters
tab tab ref The value returned by addTab(). Do not pass a string or number.
✦ Call switchTab at the very end on the tab you want shown by default when the window first opens.

minimize()

Toggles the window between collapsed (title bar only) and expanded. The built-in — button calls this too.

FN win:minimize() ▾

No parameters. Animates the window to title-bar-only height. Has a debounce — rapid clicks are ignored while the animation plays.

lua
win:minimize()

destroy()

Closes the window. Optionally shows a confirm dialog first. Saves the window position before destroying.

FN win:destroy( skipConfirm?, onDone? ) ▾
Parameters
skipConfirm optional boolean If true, destroys immediately without a dialog. Defaults to false — shows "Are you sure?" dialog.
onDone optional function Called after the window is fully destroyed.
lua
win:destroy(true) -- skip confirm
win:destroy(false, function() print("closed") end)

onClose()

Registers a callback that fires when the X button is pressed. All managed connections are disconnected and the ScreenGui is destroyed after callbacks run.

FN win:onClose( callback ) ▾
Parameters
callback function Called before the GUI is destroyed. Use to clean up loops, connections, or state.
lua
win:onClose(function()
    print("Window closed — cleaning up")
end)
04 — elements

addButton()

Adds a clickable button. Supports a cooldown timer and a disabled state.

RET ctrl = win:addButton( name, callback, options? ) ▾
Parameters
name string Label on the button. Supports text wrapping.
callback optional function Called on click.
options optional table cooldown: number — seconds before button is clickable again (shows countdown). disabled: boolean — start disabled.
Returns — ctrl object
ctrl.setText(str) — update button label
ctrl.getText() — returns current label
ctrl.setDisabled(bool) — enable or disable
ctrl.isDisabled() — returns disabled state
lua
win:addButton("Teleport to Spawn", function()
    game.Players.LocalPlayer.Character.HumanoidRootPart.CFrame = CFrame.new(0,5,0)
end, { cooldown = 3 })

addToggle()

Adds an animated pill-style on/off toggle. Supports the flag system for config persistence.

RET ctrl = win:addToggle( name, default, callback, flag? ) ▾
Parameters
name string Label shown to the left of the toggle.
default optional boolean Initial state. Defaults to false.
callback optional function(v: boolean) Fires on every state change with the new value.
flag optional string Flag key for the config system. Value is saved/loaded automatically when configName is set.
Returns — ctrl object
ctrl:set(bool, silent?) — set state; fires callback unless silent=true
ctrl:get() — returns current boolean
ctrl.Value — readable/writable, does NOT fire callback
ctrl:setDisabled(bool) — grey out and block interaction
lua
local esp = win:addToggle("ESP", false, function(v)
    print("ESP:", v)
end, "esp_on")

esp:set(true)
print(esp:get())

addCheckbox()

Same boolean logic as a toggle but rendered as a square checkbox with a checkmark. Supports the flag system.

RET ctrl = win:addCheckbox( name, default, callback, flag? ) ▾
Parameters
name string Label shown to the right of the checkbox.
default optional boolean Initial checked state. Defaults to false.
callback optional function(v: boolean) Fires on state change.
flag optional string Flag key for the config system.
Returns — ctrl object
ctrl.set(bool, silent?) — set state
ctrl.get() — returns current boolean
ctrl.setDisabled(bool) — disable interaction
lua
win:addCheckbox("Show Tracers", false, function(v)
    print("Tracers:", v)
end)

addRadioGroup()

Adds a group of radio buttons — only one option can be selected at a time.

RET ctrl = win:addRadioGroup( name, options, default, callback, flag? ) ▾
Parameters
name string Group label shown above the options. Pass "" to hide it.
options string[] Array of option strings.
default optional string Initially selected value. Defaults to options[1].
callback optional function(v: string) Fires when selection changes.
flag optional string Flag key for config persistence.
Returns — ctrl object
ctrl.get() — returns current selected string
ctrl.set(str, silent?) — set selection programmatically
lua
win:addRadioGroup("Team", {"Red", "Blue", "Green"}, "Red", function(v)
    print("Team:", v)
end)

addSlider()

Adds a draggable slider with a fill bar and live value readout. Works with both mouse and touch.

RET ctrl = win:addSlider( name, min, max, default, step, callback, flag? ) ▾
Parameters
name string Label shown above the slider.
min number Minimum value. Must be strictly less than max.
max number Maximum value.
default optional number Starting value. Clamped to [min, max]. Defaults to min.
step optional number Snap increment. Defaults to 1. If omitted, the callback can be passed in this position.
callback optional function(v: number) Fires continuously while dragging.
flag optional string Flag key for config persistence.
Returns — ctrl object
ctrl.set(num) — set value, clamped and snapped, does not fire callback
ctrl.get() — returns current value
ctrl.setDisabled(bool) — disable interaction
⚠ If min >= max the slider is silently skipped and a no-op object is returned.
lua
local spd = win:addSlider("Walk Speed", 16, 200, 16, function(v)
    game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = v
end)

local vol = win:addSlider("Volume", 0, 100, 50, 5, function(v)
    print(v)
end, "volume")

addTextBox()

Adds a labeled text input. Callback fires on Enter press. Supports a live callback for every keystroke via options.

RET ctrl = win:addTextBox( name, placeholder, maxLength?, callback, options? ) ▾
Parameters
name string Label shown above the input.
placeholder optional string Ghost text shown when empty.
maxLength optional number Max character count. Defaults to 200. If omitted, callback can be passed here.
callback optional function(v: string) Fires on FocusLost only when Enter was pressed.
options optional table liveCallback: function(v) fires on every keystroke. flag: string for config persistence. disabled: boolean.
Returns — ctrl object
ctrl.get() — returns current text
ctrl.set(str) — set text programmatically
ctrl.clear() — clears the field
ctrl.setDisabled(bool) — make read-only
lua
local box = win:addTextBox("Target Player", "Enter username...", function(v)
    print("Submitted:", v)
    box.clear()
end, { liveCallback = function(v) print(v) end })

addDropdown()

Adds a dropdown with a floating option panel. Options can be refreshed at runtime.

RET ctrl = win:addDropdown( name, options, default, callback, flag? ) ▾
Parameters
name string Label to the left of the dropdown button.
options string[] Array of option strings. Must not be empty.
default optional string Initially selected. Defaults to options[1].
callback optional function(v: string) Fires when selection changes.
flag optional string Flag key for config persistence.
Returns — ctrl object
ctrl.get() — returns currently selected string
ctrl.set(str) — programmatically select an option
ctrl.refresh(newOptions) — replace the option list entirely
ctrl.setDisabled(bool) — disable interaction
ctrl.destroy() — remove the dropdown and clean up
lua
local dd = win:addDropdown("Mode", {"Auto", "Manual"}, "Auto", function(v)
    print("Mode:", v)
end)

-- dynamically update options
local names = {}
for _, p in pairs(game.Players:GetPlayers()) do
    table.insert(names, p.Name)
end
dd:refresh(names)

addMultiDropdown()

Like a dropdown, but allows multiple selections. Returns an array of selected values.

RET ctrl = win:addMultiDropdown( name, options, defaults, callback, flag? ) ▾
Parameters
name string Label to the left of the button.
options string[] Array of option strings.
defaults optional string[] Initially selected items. Defaults to {}.
callback optional function(v: string[]) Fires with an array of selected strings every time selection changes.
flag optional string Flag key for config persistence. Stored as an array.
Returns — ctrl object
ctrl.get() — returns array of selected strings
ctrl.set(array) — set selected items programmatically
ctrl.refresh(newOptions) — replace the option list
lua
local multi = win:addMultiDropdown("Tags", {"Red", "Blue", "Green"}, {"Red"}, function(v)
    print("Selected:", table.concat(v, ", "))
end)

addColorPicker()

Adds a row with a color preview button. Clicking opens a floating HSV picker with a hex input field.

RET ctrl = win:addColorPicker( name, defaultColor, callback, flag? ) ▾
Parameters
name string Label shown to the left of the preview button.
defaultColor optional Color3 Initial color. Defaults to Color3.fromRGB(255,255,255).
callback optional function(v: Color3) Fires continuously while the color is being adjusted (SV canvas and hue bar).
flag optional string Flag key for config persistence. Stored as Color3.
Returns — ctrl object
ctrl.get() — returns current Color3
ctrl.set(color) — update color programmatically
ctrl.setDisabled(bool) — disable and close picker
ctrl.destroy() — clean up picker panel
✦ The picker panel floats above the window and auto-repositions when the window moves. It includes a hex input field that accepts formats like #FF00AA.
lua
local cp = win:addColorPicker("ESP Color", Color3.fromRGB(80,200,120), function(v)
    print("Color:", v)
end, "esp_color")

addKeybind()

Adds a keybind row. Click to enter listening mode — the next key pressed becomes the binding. Click again while listening to clear it.

RET ctrl = win:addKeybind( name, defaultKey, callback ) ▾
Parameters
name string Label. Also used as the internal registry key — must be unique within the window.
defaultKey optional Enum.KeyCode Initial binding. Pass nil for unbound.
callback optional function Called every time the bound key is pressed. Not called during listening mode.
Returns — ctrl object
ctrl.get() — returns current Enum.KeyCode or nil
ctrl.set(keyCode) — assign a key programmatically
⚠ Keybind names must be unique per window. Two keybinds with the same name will overwrite each other in the registry.
lua
win:addKeybind("Toggle Fly", Enum.KeyCode.F, function()
    print("Fly toggled")
end)

addHoldButton()

A hold-to-activate element. Active while held (mouse or key). A cosmetic progress bar fills over the configured duration while held.

RET ctrl = win:addHoldButton( name, keyCode, holdDuration, onActive, onInactive ) ▾

Unlike a toggle, this is only active while the input is physically held. Both mouse and keyboard work simultaneously — releasing either while the other is held keeps it active.

Parameters
name string Label shown on the element.
keyCode optional Enum.KeyCode Keyboard shortcut. Pass nil for mouse-only.
holdDuration optional number Seconds the progress bar takes to fill. Defaults to 1. Cosmetic only — activates on press immediately.
onActive optional function Fires once when the hold begins.
onInactive optional function Fires once when the hold ends (input released).
Returns — ctrl object
ctrl.isActive() — returns true while being held
ctrl.setActive(bool) — force state visually, does not fire callbacks
lua
win:addHoldButton("Sprint", Enum.KeyCode.LeftShift, 2,
    function() game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = 50 end,
    function() game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = 16 end
)

addPlayerList()

Adds a live player list with avatar thumbnails. Two modes depending on the type of the first argument.

RET ctrl = win:addPlayerList( labelOrCallback, callback? ) ▾
Action Mode — first argument is a string
label string Text on a button beside each player card (e.g. "Kick", "Teleport").
callback function(player) Fires with the Player instance when their action button is clicked.
Select Mode — first argument is a function
callback function(player, selected) Clicking a card toggles its selected state. Only one player can be selected at a time.
Returns — ctrl object
ctrl.refresh() — rebuild the list from current players
ctrl.getSelected() — returns selected Player or nil (select mode only)
✦ Avatars load asynchronously via GetUserThumbnailAsync so the list appears instantly and portraits fill in as they arrive.
lua — action mode
win:addPlayerList("Teleport", function(player)
    local char = player.Character
    if char then
        game.Players.LocalPlayer.Character.HumanoidRootPart.CFrame
            = char.HumanoidRootPart.CFrame
    end
end)
lua — select mode
local list = win:addPlayerList(function(player, selected)
    print(player.Name, selected)
end)
print(list.getSelected())

addStatus()

A two-column row — static label on the left, dynamically updatable value on the right with optional color.

RET ctrl = win:addStatus( name, initialValue? ) ▾
Parameters
name string Static label on the left.
initialValue optional string Starting value text. Defaults to "—".
Returns — ctrl object
ctrl.get() — returns current value string
ctrl.set(str, color?) — update value and optionally its Color3
lua
local ping = win:addStatus("Ping", "—")
task.spawn(function()
    while true do
        local ms = math.round(game:GetService("Stats").Network.ServerStatsItem["Data Ping"]:GetValue())
        local col = ms < 100 and Color3.fromRGB(80,200,120) or Color3.fromRGB(224,92,92)
        ping.set(ms .. "ms", col)
        task.wait(1)
    end
end)

06 — config & flags

Flag System

The flag system lets elements automatically save and restore their values. Pass a configName to UILibrary.new() and a flag string to any element that supports it. Values are saved to MIKEYWARE_CONFIGS/configName.json.

✦ Read current flag values at any time via win.Flags — it's a table keyed by flag name. You don't need to store every ctrl return value just to check a setting.
lua — full flag example
local win = UILibrary.new("My Script", { configName = "cfg" })

win:addToggle("ESP", false, function(v) print(v) end, "esp")
win:addSlider("Speed", 16, 200, 16, function(v) print(v) end, "speed")

-- read flags anywhere
print(win.Flags.esp)    -- true/false
print(win.Flags.speed)  -- number

-- manually save/load
win:saveConfig()
win:loadConfig()

saveConfig()

Writes all current flag values to disk. Called automatically on close when configName is set. Safe to call manually at any time.

FN win:saveConfig() ▾

Returns true on success, false if no configName was set. Saves to MIKEYWARE_CONFIGS/configName.json.

loadConfig()

Reads saved flag values from disk and applies them to all registered UI elements via applyFlags(). Called automatically on window creation when configName is set.

FN win:loadConfig() ▾

Returns the loaded data table on success, or false if no config exists yet. All elements with matching flag keys are updated immediately.

07 — misc

notify()

Shows a toast notification sliding in from the right. Multiple notifications stack automatically.

FN win:notify( title, body, imageId?, persistent? ) ▾
Parameters
title string Bold primary text. Toast header shows Mikeyware • title.
body optional string Smaller text below the title.
imageId optional number | table Roblox asset ID as a plain number, or { userId = number } to load a player headshot automatically.
persistent optional boolean If true, never auto-dismisses. Defaults to false — 6 second lifetime.
✦ Notifications live in a separate AK_NOTIF_GUI ScreenGui. A click sound plays on each appearance. Returns a nd object with nd.dismiss() to close it early.
lua
win:notify("Loaded", "Script is ready.")
win:notify("Player Joined", "Welcome!", { userId = game.Players.LocalPlayer.UserId })
win:notify("Warning", "Low FPS detected", nil, true)

confirm()

Shows a modal confirmation dialog over the window with Confirm and Cancel buttons.

FN win:confirm( message, onConfirm, onCancel ) ▾
Parameters
message string Text shown in the dialog body.
onConfirm function Called when the user clicks Confirm.
onCancel optional function Called when the user clicks Cancel.
lua
win:confirm("Reset all settings?", function()
    print("Confirmed!")
end)

addConnection()

Connects a signal and registers it so it is automatically disconnected when the window closes.

RET conn = win:addConnection( signal, callback ) ▾

Use this instead of bare signal:Connect(fn) for anything inside the window's lifecycle. All registered connections are disconnected when X is pressed.

Parameters
signal RBXScriptSignal Any Roblox signal (e.g. RunService.Heartbeat, Players.PlayerAdded).
callback function The handler to connect.
Returns
RBXScriptConnection — same as :Connect() returns
lua
win:addConnection(game:GetService("RunService").Heartbeat, function(dt)
    print(dt)
end)
08 — hud

addHUD()

Creates a standalone draggable HUD bar — completely separate from any window. Includes a player info section, live FPS/ping chips, a server browser panel, a command bar, and a settings panel.

NEW UILibrary.addHUD( options? ) ▾

Called as a static method on UILibrary, not on a window instance. Creates its own AK_HUD_GUI ScreenGui. Any existing HUD is replaced on re-call.

Options Table
theme string | table Built-in themes: "Crimson" (default), "Dark", "Slate". Or pass a custom table overriding any theme keys.
position string Starting position: "BelowChat" (default), "TopLeft", "TopRight", "BottomLeft", "BottomRight".
Returns
The hudGui ScreenGui instance.
⚠ The HUD includes a built-in server browser that makes HTTP requests to the Roblox games API. It requires game:HttpGet to be available. The command bar integrates with _G.MWCmds if loaded.
lua
-- default Crimson theme, below chat
UILibrary.addHUD()

-- Dark theme, bottom right
UILibrary.addHUD({ theme = "Dark", position = "BottomRight" })

-- Custom accent color on top of Crimson
UILibrary.addHUD({
    theme = {
        accent    = Color3.fromRGB(80,120,255),
        accentHot = Color3.fromRGB(100,150,255),
    },
})
✦ HUD sound settings (click/hover sounds and asset IDs) are saved to MIKEYWARE_CONFIGS/AK_HUD_CONFIG.json automatically.
